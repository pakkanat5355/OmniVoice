1. FIX — Echo suppression (VAD logic)
   File: voicebot/server_gpu.py
--------------------------------------------------------------------------------
ปัญหา:
  - Bot พูดแล้วเสียงตัวเองวนกลับมา (network echo) ถูก ASR แปลงซ้ำๆ
    เช่น "สวัสดีค่ะ สวัสดีค่ะ สวัสดีค่ะ..." ทำให้ ASR ช้า ~5600ms
  - สาเหตุ: ตอน bot พูด energy สูง → โค้ดเดิม set is_speaking=True
    → สะสม echo ใน audio_buf → ส่ง Whisper

แก้ไข:
  - เพิ่ม bot_is_busy flag (current_task ยังทำงาน หรืออยู่ใน cooldown)
  - ถ้า bot_is_busy และ energy สูง → clear audio_buf ทันที (suppress echo)
  - ถ้า idle และ energy ต่ำ → clear audio_buf ด้วย (ป้องกัน noise สะสม)
  - เมื่อ speech เริ่มใหม่ → เริ่ม audio_buf จาก chunk นั้นเลย ไม่เอา noise ก่อนหน้า

3. FIX — Pre-ASR echo detection
   File: voicebot/server_gpu.py
--------------------------------------------------------------------------------
ปัญหา:
  - Echo บางส่วนผ่าน VAD suppression มาได้ (delay จาก SIP trunk ~19+ วินาที)
  - ต้องรอ Whisper 5+ วินาทีก่อนจะ detect ว่าเป็น hallucination

แก้ไข:
  - เพิ่ม _is_repetitive_echo() — ก่อนส่ง Whisper ตรวจ:
      * audio ยาว > 3 วินาที
      * energy สูงสม่ำเสมอ (mean > 50)
      * coefficient of variation < 0.35 (ไม่มี natural pause)
    → ถ้าใช่: log "Pre-ASR echo discarded" แล้ว return ทันที ไม่เสีย 5+ วินาที

หมายเหตุ:
  - Echo เกิดจาก SIP trunk (Aspect proxy 172.18.72.66) ไม่ใช่ Asterisk dialplan
  - Asterisk dialplan ใช้ AudioSocket ส่ง inbound audio เท่านั้น (ไม่มีปัญหา)

6. FEAT — Text normalization ด้วย Local LLM (Qwen2.5-1.5B on CPU)
   File: voicebot/server_gpu.py
--------------------------------------------------------------------------------
แทนที่ rule-based (pythainlp) ด้วย Qwen/Qwen2.5-1.5B-Instruct บน CPU

ความสามารถ:
  - ตัวเลข → คำอ่านภาษาไทย  (เช่น 1,250 → หนึ่งพันสองร้อยห้าสิบ)
  - X%     → Thai word        (เช่น 20% → ยี่สิบเปอร์เซ็นต์)
  - English → Thai phonetic   (เช่น ABC → เอบีซี, SMS → เอสเอ็มเอส)
  - เพิ่มวรรคพักตามธรรมชาติ

การออกแบบ:
  - โหลดบน CPU → ไม่แย่ง GPU กับ TTS/ASR
  - LRU cache 512 entries → IVR phrases เดิมๆ instant (ไม่ต้อง infer ซ้ำ)
  - Async wrapper ผ่าน thread executor → ไม่ block event loop
  - Fallback กลับ original text ถ้า LLM error

ต้อง install:
  uv add accelerate

7. FEAT — server_gpu_jamai.py (ไฟล์ใหม่)
   File: voicebot/server_gpu_jamai.py
--------------------------------------------------------------------------------
สร้าง server variant ใหม่ที่ใช้ JamAI เป็น LLM backend

Flow:
  Caller → ASR (Typhoon Whisper) → JamAI API → TTS (OmniVoice) → Caller

JamAI API:
  POST http://localhost:8989/api/llm
  body: { "text": "<asr_text>" }
  extract: result.rows[0].columns.ai.choices[0].message.content

Features:
  - log executeTimeMs ทุก turn
  - Error fallback: พูด "ขออภัยค่ะ ระบบขัดข้องชั่วคราว" ถ้า API ไม่ตอบ
  - ใช้ VAD + echo suppression เดียวกับ server_gpu.py
  - ไม่มี IVR state machine — JamAI จัดการ logic ทั้งหมด
  - ไม่มี Qwen TN LLM — เบากว่า (JamAI response format Thai words อยู่แล้ว)

ต้อง install:
  uv add httpx

Run:
  uv run python voicebot/server_gpu_jamai.py
